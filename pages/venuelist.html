The problem

In this fast paced world professionals have to juggle their time in between many things, they would like a convenient way to view and book venues for their events

Prototype validation

I validated the prototype with 4 users. Each user was given the prototype to try it out. I wanted to be sure that the users understand that the users were able to navigate and view to their desired venue.

This was tested on a zoom call where I introduced the user to the app and asked them questions, the questions were dedicated in finding out whether the user was able to book their venue with ease.

Study Results

60% of the users were able to navigate through ease, but 2 users wanted to book the venue directly from the Venue information page, also the users knew that tapping on the card displays more information, but they did not expect that you can change the card layout to display more information in the card itself.

Accessibility Evaluation

The app has been evaluated for contrast to match the AA standards of WCAG. In some instances, it has been found that the contrast can be improved.

One specific example, I would like to share is the main CTA button. Initially the background of the button was selected as a lighter shade of blue, but the choice of color proved not to be fully accessible. In the final version the background color was modified

Project Summary

During this project I managed to evaluate the market, created user personas, performed a mini usability study, created user flows and set of low fidelity wireframes, connected them into a prototype and built them out to high fidelity UI designs. In the last checkup round a QA audit was done focusing on consistently and color contrast with regards to accessibility.
