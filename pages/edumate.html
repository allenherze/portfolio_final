Site Usability Analysis

As stated before, this project was a redesign of the website and before proceeding with the design phase it was better to have a look at the existing website and survey the users who can highlight the issues related to the overall experience.

The main issue with the existing website was the lack of consistency with the overall experience when viewing the website

First of all I would like to highlight about the inconsistent alignment and padding of the elements

Then on the hero landing section, the call to action button looks similar even though the actions which the users need to take are different.

The navigation links have a healthy option for navigating the website, but the choice is too many which is packed into a smaller width

And there are two chat options instead of one, which is there for contacting the company.

And even though the colors are consistent as per the branding, the overall feel of the of the website does not offer a clean and consistent experience, but the site can be improved.

Design Decisions

As stated, the main aim of the redesign was to simplify and create a new brand identity.

At the start the navigation was simplified to include four options which is used to navigate the sections of the website with two user actions of going home and logging in. The options are Subjects offered, Features of the app, Pricing of the tests, and a Contact Us link.

With ample amount of whitespace and relevant options, the navigation is made simpler to scan.

The Landing Hero section is updated with the two clear call to actions, one is a to create an account which is the get started button and to take a tour of the app button with a distinct difference between the two call to action buttons.

And the section of the sites are made consistent and with good whitespace padding so that there is a distinct start and end and the users can browse the site at ease.

Prototype Validation

I validated the prototype with 5 users. Each user was given the prototype to try it out. I wanted to be sure that the users understand that the users were able to browse the website and choose their preparation plan accordingly.

This was tested on a zoom call where I introduced the user to the website and asked questions and also told the users to mention any issues faced.

Study Results

Most of the users reported improvement in navigating the website. But three users suggested that when they click on the Get started button, they want to view and select the preparation plan and then create the account.

When a user clicks on the get started button I have added a screen where the user can choose to create the account or select the plan first and then create the account.
